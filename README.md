# 🌍 Aetheria: Earth Edition

**A cinematic, real-time 3D planetary simulator** that combines high-fidelity WebGL graphics with the intelligence of **Google Gemini AI** — all running in your browser.

Generate alien civilizations, terraform dead worlds, trigger apocalyptic events, and explore procedurally generated planet surfaces with AI-powered lore. Aetheria turns your browser into a planetary command center.

## ✨ Features

### 🌐 Real-Time 3D Rendering
- High-fidelity Earth with satellite textures, normal maps, and specular highlights
- Dynamic day/night cycle with procedural city lights that activate on the dark side
- Volumetric, animated cloud layer driven by simplex noise
- Atmospheric scattering glow with configurable color
- Orbiting satellite particle ring

### 🤖 AI-Powered World Generation
- **Generate World** — Gemini AI invents an entirely new planet with unique lore, atmosphere, and civilization status
- **Texture Synthesis** — Imagen generates equirectangular surface textures on demand
- **Probe Reports** — Click any point on the globe to receive a sci-fi location report generated by AI

### 🎮 Interactive Simulation Modes
| Mode | Description |
|---|---|
| **Terraforming** | Adjust cloud density, snow cover, toxicity, and sun type in real time |
| **Data Layers** | Toggle between Visual, Thermal, Population, and Vegetation overlays |
| **Evolution** | Fast-forward 1,000 years to see civilizations rise or collapse |
| **Disaster** | Trigger catastrophic events — gamma-ray bursts, supervolcanoes, AI takeovers |
| **Challenge** | Puzzle mode: restore a dead planet to habitability by adjusting parameters |

### 🔊 Procedural Audio
- Adaptive pink noise wind engine tied to rotation speed and cloud density
- Low-frequency oscillator hum that scales with city light intensity
- Built on the Web Audio API with AudioWorklet for efficient processing

## 🛠 Tech Stack

| Layer | Technology |
|---|---|
| **UI Framework** | React 19 with TypeScript 5.8 |
| **3D Engine** | Three.js 0.182 via React Three Fiber + Drei |

| **Audio** | Web Audio API + AudioWorklet (pink noise processor) |
| **Testing** | Vitest 4 + React Testing Library + R3F Test Renderer |
| **CI/CD** | GitHub Actions (typecheck → test → build) |
| **Server** | Node.js HTTP server (API proxy for Gemini) |

## 🚀 Installation

### 1. Clone the repository

```bash
git clone https://github.com/dhaatrik/aetheria_-earth-edition.git
cd aetheria_-earth-edition
```

### 2. Install dependencies

```bash
npm install
```

### 3. Configure your API key

Create a `.env` file in the project root:

```env
GEMINI_API_KEY=your_actual_api_key_here
```

### 4. Start the application

You need **two terminals** — one for the API server and one for the frontend:

**Terminal 1 — API Server:**
```bash
npm run server
```
> Starts the Gemini proxy server at `http://127.0.0.1:3001`

**Terminal 2 — Frontend:**
```bash
npm run dev
```
> Opens the app at `http://localhost:3000`

## 🎮 Usage

Once the app loads, you'll see a photorealistic Earth floating in space.

### Controls

| Action | How |
|---|---|
| **Rotate** | Click and drag on the background |
| **Zoom** | Mouse scroll wheel |
| **Scan Location** | Click anywhere on the planet surface |
| **Generate World** | Top-right button — AI creates a new planet |
| **Photo Mode** | Hides all UI for clean screenshots |
| **Toggle Audio** | Enables/disables the procedural soundscape |

### Dashboard Panel (Bottom-Left)

- **Tabs** — Switch between `Visual`, `Thermal`, `Pop` (city lights), and `Bio` (vegetation) data layers
- **Sliders** — Adjust cloud density, snow cover (ice age), and toxicity in real time
- **Star System** — Change the sun type between Yellow, Red Dwarf, and Blue Giant
- **Buttons** — `Evolution Step`, `Trigger Event`, `Start Terraforming Challenge`
- **Toggles** — Show/hide satellites, enter/exit probe landing mode

### Lore Panel (Bottom-Right)

Displays the current planet's name, description, civilization type, and habitability score.

## ⚠️ Troubleshooting

**"I click Generate and nothing happens"**
- Verify your `.env` file has a valid `GEMINI_API_KEY`
- Ensure the API server is running (`npm run server`) in a separate terminal
- Restart both servers after changing the `.env` file

**"The planet is black or invisible"**
- Ensure your browser supports WebGL (all modern browsers do)
- Check that hardware acceleration is enabled in your browser settings

**"Tests are failing"**
- Run `npm install` to ensure all devDependencies are installed
- Run `npm run typecheck` to check for TypeScript errors
- The `act(...)` warnings in test output are expected in jsdom environments
